Without SpecFeature

The solution, step‑by‑step

A guided, logical flow that mirrors how real teams build: define → design → deliver → verify.

Describe once

Use spec/feature.md and pass @<feature>@ <context>. This becomes your single source of truth.

Use the template from spec/feature.md.
@user-auth@ Users register, login, reset password; integrate with notifications.

Generate artifacts

AI creates three Markdown docs in /spec/features/<feature>: spec, plan, tasks.

spec/features/user-auth/
├── spec.md   # value & scenarios
├── plan.md   # technical design
└── tasks.md  # implementation checklist

Execute & verify

Work through tasks; then run verify.md to compare implementation against the spec and plan. A verify-report.md is produced.

Execute all tasks in spec/features/user-auth/tasks.md
→ Run verification via spec/core/verify.md
→ Get verify-report.md with ✅ / ❌ and recommendations

Install in seconds

Use npx or install globally/locally.

Using npx (recommended)

# Initialize in current directory
npx spec-feature init

# Custom folder name
npx spec-feature init my-project-docs

Global install

npm install -g spec-feature

# Then use it anywhere
spec-feature init
spec-feature init my-features

Local install

npm install spec-feature

# package.json
{
  "scripts": {
    "init-spec": "spec-feature init",
    "init-docs": "spec-feature init documentation"
  }
}

FAQ

How do I start a new feature?

Open spec/feature.md in your AI assistant and pass @<feature>@ <context>.

Can I update specs later?

Yes. Reuse spec/feature.md with the same feature name; we regenerate spec/plan/tasks and keep structure intact.

What triggers verification?

When tasks are checked as complete, run spec/core/verify.md. It produces verify-report.md with ✅ / ❌ and recommendations.

Which assistants are supported?

Any that can create files/folders: Cursor, Claude Code, Copilot, Gemini CLI, etc.
